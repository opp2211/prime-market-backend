# AGENTS.md

- Всегда отвечай пользователю на русском языке.
- Перед началом задачи изучи связанные контроллеры, DTO, сервисы, репозитории, миграции и тесты.
- Не переписывай архитектуру без необходимости; встраивайся в существующую модель.
- ripgrep (`rg`) отсутствует в окружении: сразу используй доступные штатные средства (`git grep`, `grep`, `find`, PowerShell/IDE search).
- При изменении API по возможности сохраняй обратную совместимость; если меняешь контракт, обновляй read-model и тесты согласованно.
- Для исторических записей предпочитай snapshot-поля, а не зависимость от live-справочников.
- Для денежных, статусных и конкурентных операций учитывай транзакционность, блокировки, идемпотентность и audit trail.
- Для изменений БД используй новые миграции, не редактируй старые без прямого указания.
- Перед сдачей прогоняй релевантные проверки и не заявляй об успехе без проверки.
- После успешного выполнения каждой законченной задачи и прохождения релевантных проверок обязательно делай `git commit`.
- Сообщение коммита должно быть на русском языке и соответствовать Conventional Commits.
- В конце задачи давай короткий отчёт: изменённые файлы, что сделано, что проверено, закоммичены ли изменения, hash и текст коммита, какие ограничения остались.
